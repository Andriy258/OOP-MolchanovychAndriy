# IndependentWork13 — Polly / Retry / CircuitBreaker / Timeout

## Тема
Кейси Polly/Retry: короткий звіт.

## Мета
Дослідити реальні сценарії використання бібліотеки **Polly** у .NET для забезпечення  
відмовостійкості застосунків через політики:

- **Retry**  
- **Circuit Breaker**  
- **Timeout**  
- **Fallback**

---

# 1. Сценарій 1 — Зовнішній API (Retry)

### Проблема
Зовнішній HTTP-API може тимчасово бути недоступним (мережеві помилки, проблеми інфраструктури).  
Такі збої зазвичай короткочасні, тому повторна спроба часто проходить успішно.

### Обрана політика Polly
**Retry + експоненційна затримка**  
Причина: збій тимчасовий, повторна спроба має високий шанс успіху.

### Імітація
Метод `CallExternalApi()` у перших викликах кидає `HttpRequestException`.  
Політика `WaitAndRetry()` повторює до 3 разів: 2s → 4s → 8s.

### Очікування
- Перші 2 запити — помилка  
- На 3-й — успішна відповідь  
- Усі спроби та помилки логуються

---

# 2. Сценарій 2 — Доступ до Бази Даних (Retry + CircuitBreaker)

### Проблема
Підключення до БД може нестабільно працювати (тимчасові збої, перезапуск сервера).  
Також під час частих помилок варто перекрити запити, щоб не перевантажити БД.

### Обрана політика Polly
- **Retry** — для поодиноких збоїв  
- **CircuitBreaker** — при серії помилок «перекриває» доступ на певний час

### Імітація
Метод `QueryDatabase()` кілька разів кидає `InvalidOperationException`.

### Очікування
- Retry робить повтори з затримками  
- Circuit Breaker відкривається після 2 помилок  
- Усі події breaker логуються:
  - break
  - half-open
  - reset

---

# 3. Сценарій 3 — Довга операція (Timeout + Fallback)

### Проблема
Деякі операції можуть виконуватись занадто довго або зависати.

### Обрана політика Polly
- **Timeout** — жорстке обмеження часу  
- **Fallback** — контрольований результат замість аварійного падіння

### Імітація
`LongRunningOperation()` виконується 5 секунд.  
Timeout = 2 секунди → операцію буде перервано.

### Очікування
- Timeout перериває операцію  
- Fallback повертає безпечний результат  
- Логування фіксує спрацювання Timeout і Fallback

---

# Висновки

- **Polly** дозволяє просто й декларативно додавати відмовостійкість до .NET застосунків.  
- **Retry** підходить для тимчасових помилок, але потребує обережності, щоб не перевантажити ресурс.  
- **CircuitBreaker** запобігає масовим запитам до нестабільного сервісу.  
- **Timeout** не допускає зависання потоків.  
- **Fallback** забезпечує контрольований вихід із помилки.  
- Комбінація політик дозволяє створювати складні механізми обробки помилок у реальних системах.
